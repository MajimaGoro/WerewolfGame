# 双身份狼人杀自动主持

一个适合线下聚会的单手机狼人杀网页主持工具。

这个项目的目标不是替大家在线统计票型，而是让网页承担“身份发放、固定语音播报、夜晚倒计时、结果录入、赛后复盘”的工作，让所有人都能参与游玩，不需要额外真人 DM。

项目适合部署到 GitHub Pages，并支持基础 PWA 离线安装与本地存档。

## 当前玩法定位

当前版本更贴近你们现在的线下实际玩法：

- 单手机轮流操作
- 网页负责主持和记录
- 夜晚按固定顺序播报与倒计时
- 白天投票在线下自行统计
- 网页只录入最终结果
- 夜晚通过固定时长与缓冲，避免暴露场上是否存在某个角色

这意味着网页不会替玩家统计白天票型，也不会通过跳过夜晚环节来泄露身份信息。

## 功能概览

- 双身份随机发放
- 单手机依次查看身份
- 无真人 DM 的自动主持流程
- 浏览器语音播报
- 预设板子与自定义板子
- 狼人讨论时间、其他角色时间可配置
- 入夜等待时间、角色切换等待时间可配置
- 白天讨论计时器
- 白天结果手动录入
- 本地自动存档
- 导入导出对局存档
- 公开日志与隐藏日志复盘
- GitHub Pages 自动部署
- PWA 离线安装

## 当前角色

- 狼人
- 预言家
- 守卫
- 女巫
- 村民

## 当前规则说明

### 双身份

- 每位玩家拥有两个身份
- 只要任一存活身份属于狼人阵营，该玩家当前按狼人阵营结算
- 每次死亡只会失去一个身份
- 默认先失去“上身份”，确认身份时可以交换上身份和下身份
- 两个身份都死亡后，玩家才算真正出局
- 真正出局后，不能继续发言、投票，也不能再成为夜晚操作目标

### 夜晚顺序

夜晚固定按以下顺序执行：

`狼人 -> 预言家 -> 守卫 -> 女巫`

说明：

- 不会因为场上没有对应角色而跳过某个阶段
- 点击“开始夜晚流程”后，才会播报“天黑请闭眼”
- “请闭眼”和下一位“请睁眼”之间会保留可配置缓冲时间
- 所有夜晚阶段都按预设倒计时自动推进，不依赖玩家是否点击来判断角色是否存在

### 各角色当前规则

狼人：
- 可在线下讨论后，在网页中选择本夜击杀目标
- 支持自刀

预言家：
- 可查验一名当前仍存活的玩家
- 查验结果显示该玩家当前阵营

守卫：
- 可守护任意一名当前仍存活的玩家
- 支持自守

女巫：
- 有一瓶解药和一瓶毒药
- 解药只能阻止本夜狼刀造成的那一个身份损失
- 毒药只能使用一次
- 毒药已使用后，页面不再显示毒人目标
- 当前版本下，女巫不能毒自己

### 白天结算

- 白天讨论结束后，线下自行统计投票
- 网页只录入“谁失去一个身份”或“无人出局”
- 不记录票型过程
- 白天播报结果时，不展示死亡原因

### 语音播报

- 默认开启语音播报
- 语音只播报阶段提示，例如“天黑请闭眼”“狼人请睁眼”“狼人请闭眼”
- 不播报查验结果、死亡原因、具体身份归属或阵营细节
- 语音不会因倒计时每秒刷新而重复从头播放

## 本地开发

要求：

- Node.js 22 或更高版本
- npm 11 或更高版本

启动开发环境：

```bash
npm install
npm run dev
```

构建生产版本：

```bash
npm run build
```

本地预览生产构建：

```bash
npm run preview
```

## 使用方式

1. 打开首页，选择预设板子或切到自定义板子。
2. 设置玩家人数、玩家名称、夜晚时间配置和身份数量。
3. 开始新对局，让玩家按座位顺序依次查看自己的双身份。
4. 玩家如有需要，可在确认前交换上身份和下身份顺序。
5. 点击开始夜晚流程后，系统才会播报“天黑请闭眼”，随后按固定顺序进入各角色阶段。
6. 夜晚阶段按固定倒计时推进，角色闭眼和下一位睁眼之间会自动保留等待时间。
7. 白天讨论结束后，线下先统计票型，再在网页中录入失去一个身份的玩家或“无人出局”。
8. 对局结束后，可查看公开日志；需要复盘夜晚细节时，再打开隐藏日志。

## 语音播报说明

- 语音播报使用浏览器原生 `SpeechSynthesis API`
- 默认开启语音播报
- 大多数浏览器仍需要你先点击一次页面按钮，后续语音才能正常播放
- 当前版本只播报阶段提示
- 当前版本不会语音播报玩家阵营、查验结果、死亡原因或身份详情

## PWA 与离线说明

项目已包含：

- `manifest.webmanifest`
- `service worker`
- 基础应用图标

使用建议：

- 首次使用时先联网打开一次站点
- 页面加载完成后，再将网页安装到桌面或主屏幕
- 之后即使现场网络不稳定，也能依靠缓存继续打开应用
- 对局数据保存在浏览器本地存储中，重要对局建议导出 JSON 备份

常见安装方式：

- Chrome / Edge 桌面端：地址栏右侧“安装应用”
- Android Chrome：浏览器菜单中的“添加到主屏幕”

## GitHub Pages 发布

仓库已包含工作流文件：

- [.github/workflows/deploy.yml](./.github/workflows/deploy.yml)

发布步骤：

1. 将仓库推送到 GitHub。
2. 打开仓库的 `Settings -> Pages`。
3. 在 `Build and deployment` 中选择 `GitHub Actions` 作为来源。
4. 推送到 `main` 分支后，GitHub Actions 会自动构建并发布站点。
5. Pages 会生成一个 `https://<username>.github.io/<repo>/` 地址。

## 数据存储与注意事项

- 对局状态默认保存在 `localStorage`
- 刷新页面后会尝试恢复上次未结束的对局
- 清空浏览器站点数据会导致本地存档丢失
- 更换浏览器或设备后，本地存档不会自动同步
- 建议长局或重要对局中途手动导出 JSON 备份
