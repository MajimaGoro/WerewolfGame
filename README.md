# 双身份狼人杀自动主持

一个适合线下聚会的单手机狼人杀网页主持工具。

这版的设计目标不是替大家在线投票，而是让网页承担“固定顺序播报、阶段倒计时、结果记录、赛后复盘”的工作，让现场所有人都能参与游戏，不需要额外的真人 DM。

项目适合部署到 GitHub Pages，并支持基础离线缓存与安装到桌面或主屏幕。

## 当前玩法定位

当前版本更贴近线下真实流程，定位是：

- 单手机轮流操作
- 网页负责主持和记录
- 夜晚按固定阶段倒计时推进
- 白天投票在线下自行统计
- 网页只录入最终结果

这意味着网页不会替玩家统计白天票型，也不会通过是否跳过某个夜晚环节来暴露场上有没有对应角色。

## 功能概览

- 双身份随机发放
- 单手机依次查看身份
- 无真人 DM 的自动主持流程
- 浏览器语音播报
- 预设板子和自定义板子
- 狼人讨论时间与其他角色操作时间可配置
- 白天讨论计时器
- 白天结果手动录入
- 本地自动存档
- 导入导出对局存档
- 赛后公开日志与隐藏日志复盘
- GitHub Pages 自动部署
- PWA 基础安装与离线缓存

## 当前角色

- 狼人
- 预言家
- 守卫
- 女巫
- 村民

## 当前规则范围

- 每位玩家拥有两个身份
- 只要任一身份属于狼人阵营，该玩家整体按狼人阵营结算
- 夜晚固定按 `狼人 -> 预言家 -> 守卫 -> 女巫` 顺序执行
- 每个夜晚阶段都按预设倒计时进行，不会因为场上没有对应角色而跳过
- 狼人阶段在手机上直接选择本夜击杀目标
- 预言家阶段直接选择查验目标
- 守卫阶段直接选择守护目标
- 女巫阶段可根据本夜击杀目标选择是否使用解药，也可选择毒人或跳过
- 白天讨论结束后，由线下自行统计投票结果
- 网页只录入“谁出局”或“无人出局”
- 白天播报死亡玩家时，不展示死亡原因
- 语音只播报阶段提示，例如“狼人请睁眼”“预言家请睁眼”
- 语音不播报查验结果、阵营信息或死亡原因
- 复盘日志按时间正序展示，例如第一夜、第一天、第二夜

## 本地开发

要求：

- Node.js 22 或更高版本
- npm 11 或更高版本

启动开发环境：

```bash
npm install
npm run dev
```

构建生产版本：

```bash
npm run build
```

本地预览生产构建：

```bash
npm run preview
```

## 使用方式

1. 打开首页，选择预设板子或切到自定义板子。
2. 输入玩家人数、玩家昵称、狼人讨论时间和其他角色时间。
3. 开始新对局，让玩家按座位顺序依次查看自己的双身份。
4. 进入夜晚后，网页会固定播报狼人、预言家、守卫、女巫阶段。
5. 每个夜晚阶段按倒计时自动推进，不会提前跳过。
6. 白天讨论结束后，线下先统计票型，再在网页中录入出局玩家或“无人出局”。
7. 对局结束后查看公开日志，必要时打开隐藏日志复盘。

## 语音播报说明

- 语音播报使用浏览器原生 `SpeechSynthesis API`
- 默认开启语音播报
- 大多数浏览器仍需要先点击一次页面按钮，后续语音才能正常播放
- 当前版本只播报阶段提示
- 当前版本不会语音播报玩家阵营、查验结果、死亡原因

## PWA 与离线说明

项目已包含：

- `manifest.webmanifest`
- `service worker`
- 基础应用图标

使用建议：

- 首次使用时先联网打开一次站点
- 页面加载完成后，再将网页安装到桌面或主屏幕
- 之后即使现场网络不稳定，也能依靠缓存资源继续打开应用
- 对局数据保存在浏览器本地存储中，建议重要对局中途导出 JSON 备份

常见安装方式：

- Chrome / Edge 桌面端：地址栏右侧“安装应用”
- Android Chrome：浏览器菜单中的“添加到主屏幕”

## GitHub Pages 发布

仓库已包含工作流文件：

- [.github/workflows/deploy.yml](./.github/workflows/deploy.yml)

发布步骤：

1. 将仓库推送到 GitHub。
2. 打开仓库的 `Settings -> Pages`。
3. 在 `Build and deployment` 中选择 `GitHub Actions` 作为来源。
4. 推送到 `main` 分支后，GitHub Actions 会自动构建并发布站点。
5. Pages 会生成一个 `https://<username>.github.io/<repo>/` 地址。

## 数据存储与注意事项

- 对局状态默认保存在 `localStorage`
- 刷新页面后会尝试恢复上次未结束的对局
- 清空浏览器站点数据会导致本地存档丢失
- 更换浏览器或设备后，本地存档不会自动同步
- 建议长局或重要局中途手动导出 JSON 备份
