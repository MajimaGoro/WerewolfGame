# 双身份狼人杀自动主持

一个适合线下聚会的单手机狼人杀网页主持工具。  
目标场景是：没有真人 DM，所有人都是玩家，网页本身负责发身份、夜晚唤醒、语音播报、白天计时、投票结算与赛后复盘。

当前版本适合部署到 GitHub Pages，并支持基础离线缓存与作为桌面应用安装。

## 功能概览

- 单手机轮流操作
- 双身份随机发放
- 无真人 DM 的自动主持流程
- 浏览器语音播报
- 预设板子和自定义板子
- 白天讨论计时器
- 本地自动存档
- 导入导出对局存档
- 赛后公开日志与隐藏日志复盘
- GitHub Pages 自动部署
- PWA 基础安装与离线缓存

## 当前角色与规则范围

当前版本内置角色：

- 狼人
- 预言家
- 守卫
- 女巫
- 村民

当前版本的关键规则约束：

- 每位玩家拥有两个身份
- 只要任一身份属于狼人阵营，该玩家整体按狼人阵营结算
- 夜晚按守卫 -> 预言家 -> 狼人 -> 女巫顺序执行
- 多个狼人身份各自投票，平票则狼刀失效
- 女巫当前版本每晚只支持三选一：救人、毒人、跳过
- 白天投票平票时无人出局

这是一版偏产品原型的实现，适合先在线下跑通流程，再逐步扩展更复杂的正式规则。

## 本地开发

要求：

- Node.js 22 或更高版本
- npm 11 或更高版本

启动开发环境：

```bash
npm install
npm run dev
```

构建生产版本：

```bash
npm run build
```

本地预览生产构建：

```bash
npm run preview
```

## 使用方式

1. 打开首页，选择预设板子或切换到自定义板子。
2. 输入玩家人数和玩家昵称。
3. 开始新对局，让玩家按座位顺序依次查看自己的双身份。
4. 网页会自动推进夜晚和白天流程。
5. 白天阶段可使用讨论计时器控场。
6. 结束后在结果页查看公开日志，必要时切换到隐藏日志做复盘。

## 语音播报说明

- 语音播报使用浏览器原生 `SpeechSynthesis API`
- 多数浏览器需要先点击一次页面按钮，语音才能开始播放
- 不同浏览器和系统的中文语音质量会不同
- 若语音不可用，页面上的文字提示仍然可以完整主持流程

## PWA 与离线说明

项目已包含：

- `manifest.webmanifest`
- `service worker`
- 基础应用图标

使用建议：

- 首次使用时先联网打开一次站点
- 等页面加载完成后，再将网页安装到桌面或主屏幕
- 之后即使网络不稳定，仍可以依赖缓存资源继续打开应用
- 对局数据保存在浏览器本地存储中，建议重要对局随时导出 JSON 备份

浏览器安装方式通常是：

- Chrome / Edge 桌面端：地址栏右侧的“安装应用”
- Android Chrome：浏览器菜单里的“添加到主屏幕”

## GitHub Pages 发布

仓库已经包含工作流文件：

- [.github/workflows/deploy.yml](./.github/workflows/deploy.yml)

发布步骤：

1. 将仓库推送到 GitHub。
2. 打开仓库的 `Settings -> Pages`。
3. 在 `Build and deployment` 中选择 `GitHub Actions` 作为来源。
4. 推送到 `main` 分支后，GitHub Actions 会自动构建并发布站点。
5. 发布完成后，Pages 会生成一个 `https://<username>.github.io/<repo>/` 地址。

如果你是第一次启用 Pages，建议同时检查：

- 仓库是否允许 GitHub Actions 运行
- Pages 来源是否确实选择了 `GitHub Actions`
- Actions 构建日志里是否有 `Deploy to GitHub Pages` 成功记录

## 数据存储与注意事项

- 对局状态默认保存在 `localStorage`
- 刷新页面后会尝试恢复上次未结束的对局
- 清空浏览器站点数据会导致本地存档丢失
- 更换浏览器或设备后，本地存档不会自动同步
- 建议长局或重要局中途手动导出 JSON 备份
